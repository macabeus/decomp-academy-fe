# Reading a three-operand instruction

Three operands. That is the shape of every PowerPC arithmetic instruction. Take
`add rD, rA, rB`, which means `rD = rA + rB`. The part that catches everyone is
the order: destination up front, the two sources trailing behind. Mix it up once
and you tend not to forget again.

Two `int` arguments? The ABI lines them up left-to-right in `r3` and `r4`. So go
stare at the target for a second. What expression over those two would the
compiler boil down to a single `add` that touches both of them?

## Your task

Write `add2`, taking two `int`s, to reproduce the target assembly.

<!-- starter -->
```c
int add2(int a, int b) {
    return 0;
}
```

<!-- solution -->
```c
int add2(int a, int b) {
    return a + b;
}
```
