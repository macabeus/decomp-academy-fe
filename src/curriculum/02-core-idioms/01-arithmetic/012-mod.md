# Remainder = divide, multiply back, subtract

PowerPC has no remainder instruction. Instead, the compiler synthesizes it from
three operations: an integer divide, a multiply, and a subtract. For two
variables `p` and `q`:

```asm
divw  r0, r3, r4   # quotient = p / q
mullw r0, r0, r4   # quotient * q
subf  r3, r0, r3   # p - (quotient * q)
blr
```

Watch the operand order on `subf`: `subf rD, rA, rB` computes `rD = rB - rA`,
the *reverse* of how the mnemonic reads. Here `subf r3, r0, r3` gives `r3 - r0`.
That backwards operand order is a classic PowerPC gotcha.

Spotting this divide → `mullw` → `subf` trio in the wild tells you the original
C used the `%` operator. The same three-instruction sequence appears regardless
of the variable names involved.

## Your task

Write `mod2` for signed `int`s to reproduce the assembly above.

<!-- starter -->
```c
int mod2(int a, int b) {
    return 0;
}
```

<!-- solution -->
```c
int mod2(int a, int b) {
    return a % b;
}
```
