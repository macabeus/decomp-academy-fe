# Arithmetic shift right by 13

On a *signed* value, `x >> 13` preserves the sign bit, so MWCC uses the
algebraic shift **`srawi r3, r3, 13`** — a real opcode, not an `rlwinm`
mnemonic. (An unsigned `>>` would zero-fill with `srwi` instead; the type, not
the operator, decides.)

## Your task
Write `sar` on an `s32` so it compiles to the `srawi` above.

<!-- starter -->
```c
s32 sar(s32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
s32 sar(s32 x) {
    return x >> 13;
}
```
